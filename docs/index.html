TurboSSH

SSH, serial, and SFTP for people who live on embedded and automotive boxes. One toolkit — a Python library, a CLI, and a desktop GUI with a real VT100 terminal.

$ pip install turbossh
Download the GUI (Windows) View on PyPI

What it does

Everything below is available from the library, the command line, and the GUI.

🔌

SSH, jump hosts, legacy gear

Connect directly, through an RDP/Windows jump box, or to old ECUs that only speak deprecated crypto. Typed results — exit code, stdout, timing.

📜

Live logs

slog2info -w, journalctl -f, dmesg -w streamed line by line, with regex matching and tee-to-file.

📁

SFTP & SCP

A full remote-filesystem API and a graphical browser under every session. Transfers run on their own channel, so nothing blocks.

🖥️

Native serial over RDP

Open a COM port that's plugged into the remote Windows box and type into it like a real terminal — Tab-completion, Ctrl-C, the lot.

🔎

Scan remote ports

Ask the machine you're about to connect to which COM ports it actually has, with friendly device names.

🖧

Offline OpenSSH install

Stand up sshd on a locked-down Windows box with no internet — on the machine, or to a remote one over WinRM.

🚇

Port forwarding

ssh -L / ssh -R tunnels through the same connection or jump host.

🪟

The GUI

Tabbed sessions, split view, SFTP browser, log viewer, 10k-line scrollback, and save-the-whole-session-to-disk.

🔐

Credentials in the vault

Passwords live in the OS keyring and are masked in logs — never in plaintext in your scripts or on disk.

Quick start

The same object works as a library in a test framework or a one-off script.

# Python — in a test framework
from turbossh import SSHHandler, SSHConfig

with SSHHandler(SSHConfig(
        host="192.168.1.50",
        username="root", password="…")) as ssh:
    print(ssh.run("uname -a").text)
    ssh.push("build/app", "/tmp/app")
    ssh.serial_stream("COM4", on_line=print)
# CLI — in a shell script
$ turbossh run    --host H --user U uname -a
$ turbossh stream --host H --user U slog2info -w
$ turbossh serial-ssh --host H --user U \
      --device COM4 --baud 115200 --save log.txt
$ turbossh scan-ports --host H --user U

# or just launch the desktop app
$ turbossh-gui

Get it

Pick whichever fits how you work.

pip

Library + CLI + the GUI launcher.
pip install turbossh then turbossh-gui.

Windows .exe

The GUI as a single self-contained executable — download the latest release, no Python needed.
